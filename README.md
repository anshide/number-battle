# Number Battle 🎮

A real-time multiplayer logic and deduction game built with a modern web stack. Challenge a friend to crack your secret number before they crack yours!

## Features 🚀

- **Real-Time Multiplayer**: Instantaneous turn-based gameplay powered by Socket.IO.
- **Customizable Game Settings**: Play with 3, 4, or 5 digit numbers, and choose whether to allow duplicate digits.
- **Session Tracking**: Track wins, losses, and draws across rematches seamlessly.
- **Private Notes**: Integrated scratchpad to jot down your deductions without leaving the game.
- **Lifetime Statistics**: Track your overall games played and win rate across sessions.
- **Beautiful UI/UX**: Premium dark mode design with glassmorphism, dynamic gradients, and fluid micro-animations.

## Technology Stack 💻

This project uses an NPM Workspaces monorepo structure:

- **Frontend (Client)**: React 18, TypeScript, Vite, TailwindCSS, React Router, Socket.IO Client.
- **Backend (Server)**: Node.js, Express, TypeScript, Socket.IO.
- **Shared**: Common TypeScript types, interfaces, and constants.

## Getting Started 🛠️

### Prerequisites

- [Node.js](https://nodejs.org/) (v18 or higher recommended)
- `npm` (comes with Node.js)

### Installation

1. Clone the repository:
   ```bash
   git clone https://github.com/anshide/number-battle.git
   cd number-battle
   ```

2. Install dependencies for all workspaces:
   ```bash
   npm install
   ```

### Running Locally

You can run both the frontend and backend concurrently from the root directory:

```bash
# Terminal 1: Start the backend server (Runs on port 3001)
npm run dev:server

# Terminal 2: Start the frontend client (Runs on port 5173)
npm run dev:client
```

Open your browser and navigate to `http://localhost:5173`. 

To play against yourself for testing, simply open an Incognito window or a different browser and join the room code generated by your first tab.

## How to Play 🧠

1. **Create a Room**: One player creates a room and receives a unique 6-character room code.
2. **Join the Room**: The second player joins using the code.
3. **Setup Phase**: Both players secretly choose a number based on the room's settings (e.g., a 4-digit unique number).
4. **Guessing**: Players take turns guessing the opponent's number.
5. **Clues**: After each guess, you will receive feedback:
   - **Correct Digits**: How many digits from your guess are present in the opponent's number.
   - **Position Correct**: How many of those digits are in the exact right spot.
6. **Winning**: Use deduction to find the exact number. The first player to get all digits in the correct position wins! If both solve it in the same round, it's a draw.
